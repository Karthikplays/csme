Introduction to Hash Tables

Hash tables are fundamental data structures that provide efficient data retrieval through key-value mapping.

Key Concepts:

  • Hash Function: Maps keys to array indices
  • Load Factor: Ratio of stored elements to table size
  • Collision: When two keys hash to the same index
  • Performance: Average O(1) for search, insert, delete
Problem: As more elements are inserted, collisions increase and performance degrades!

What is Rehashing?

Rehashing is the process of creating a new, larger hash table and redistributing all existing elements into it using a new hash function.

Why Do We Need Rehashing?

  • Maintain Performance: Keep operations at O(1) average time
  • Reduce Collisions: More space means fewer conflicts
  • Optimal Load Factor: Typically trigger when load factor > 0.7
  • Dynamic Growth: Adapt to data size requirements

Load Factor = Number of Elements / Table Size

When load factor exceeds threshold (commonly 0.7), rehashing is triggered!

When to Trigger Rehashing?

Typical Scenarios:

  • Load Factor Threshold: Most common trigger (λ > 0.7)
  • Fixed Intervals: After N insertions
  • Performance Degradation: When operations become slow
  • Manual Trigger: Application-specific requirements

New Table Size:

Typically double the current size and find the next prime number:

  • Old size: 7 → New size: 17
  • Old size: 17 → New size: 37
  • Prime numbers reduce clustering and collisions

The Rehashing Process

Step-by-Step:

  1. Detect: Check if load factor exceeds threshold
  2. Allocate: Create a new, larger hash table
  3. Transfer: Recompute hash for each element
  4. Insert: Place elements in new positions
  5. Replace: Old table is replaced by new table

Time Complexity: O(n) where n is the number of elements

Amortized Cost: O(1) per insertion over time

Rehashing Pseudo Code

function insert(key, value): // Check if rehashing is needed if (size / capacity) > LOAD_FACTOR_THRESHOLD: rehash() // Insert the key-value pair index = hashFunction(key) % capacity table[index] = (key, value) size++ function rehash(): oldTable = table oldCapacity = capacity // Create new table with double size capacity = nextPrime(capacity * 2) table = new Array[capacity] size = 0 // Transfer all elements to new table for each element in oldTable: if element is not null: insert(element.key, element.value)

Real-World Applications

Where Rehashing is Critical:

  • Database Indexing: Maintaining fast query performance
  • Caching Systems: Redis, Memcached use dynamic hash tables
  • Symbol Tables: Compilers and interpreters
  • Network Routing: IP address lookup tables
  • Spell Checkers: Dictionary implementations
  • Programming Languages: Python dicts, Java HashMaps

Example: Python's dictionary automatically rehashes when load factor exceeds 2/3 (approximately 0.67)

Conclusion

Key Takeaways:

  • Rehashing maintains O(1) performance in hash tables
  • Triggered when load factor exceeds threshold (typically 0.7)
  • Creates larger table and redistributes all elements
  • Amortized cost is O(1) despite O(n) rehashing operation
  • Essential for dynamic data structures in real applications

Remember:

Rehashing is a trade-off between space and time efficiency. It sacrifices temporary performance during rehashing to maintain long-term optimal performance.
